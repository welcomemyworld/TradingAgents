# Future Invest GitHub Launch Checklist

## Release Surface

- README top section is hook-first, quickstart-first, and comparison-first
- `Future Invest` is framed as an institutional operating system, not a stock picker
- Optional OpenAI-compatible backends are documented, but not presented as the primary path
- Proposal-style docs remain in the repo, but are not the first thing a new visitor needs

## Product Proof

- One static product screenshot is visible near the top of the README
- The recommended lean path is explicitly documented
- A known-good configuration is included for first-time users

## Repo Hygiene

- Worktree is clean before push
- Published smoke tests pass
- Generated memory, traces, and evaluation outputs are not tracked
- Release branch contains only changes intended for public launch

## Distribution

- README finalized
- Launch copy draft reviewed
- One stable showcase run is ready to reference if someone asks for proof
- Repo description and social post use the same one-line hook
